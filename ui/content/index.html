      🚀 Primeros Pasos
      1. Preparación Inicial
      Antes de usar tu USB Army Knife:
      • Asegúrate de que el dispositivo esté completamente cargado
      • Verifica que tengas una tarjeta microSD insertada (si tu dispositivo la soporta)
      • Ten listos los scripts DuckyScript que planeas usar
      • Considera el entorno donde vas a realizar las pruebas
      2. Primera Conexión
      Proceso paso a paso:
      1. Conecta el dispositivo: Inserta el USB Army Knife en cualquier puerto USB
      2. Espera la inicialización: El dispositivo necesita 10-15 segundos para arrancar completamente
      3. Busca la red WiFi: Aparecerá una red llamada "iPhone14" (o el nombre que hayas configurado)
      4. Conéctate a la red: Usa la contraseña "password" (por defecto)
      5. Abre tu navegador: Ve a la dirección http://4.3.2.1:8080

      📝 Creando y Ejecutando Scripts
      Subir un Script DuckyScript

      Método 1: A través de la interfaz web

      1. Ve a la pestaña "Archivos"
      2. Usa el explorador de archivos para navegar por la SD
      3. Sube archivos .ds (DuckyScript) usando el botón de carga

      Método 2: Directamente en la SD

      1. Retira la tarjeta microSD del dispositivo
      2. Conéctala a tu computadora
      3. Copia los archivos .ds directamente
      4. Vuelve a insertar la SD en el dispositivo
      Ejecutar un Ataque

      Ejecución Manual:

      1. En la pestaña "Archivos", selecciona tu script .ds
      2. Haz clic en "Execute" para ejecutarlo inmediatamente
      3. Observa los logs en la pestaña "Registros" para ver el progreso

      Ejecución Automática:

      1. Renombra tu script a "autorun.ds"
      2. Colócalo en la raíz de la tarjeta SD
      3. El script se ejecutará automáticamente al conectar el dispositivo
      🔧 Modos de Operación
      Modo HID (Teclado/Ratón)

      ¿Qué hace? El dispositivo se comporta como un teclado y ratón, permitiendo inyectar comandos.

      Casos de uso:

      • Inyección de teclas para automatizar tareas
      • Apertura de terminales y ejecución de comandos
      • Navegación por menús y aplicaciones
      • Ataques BadUSB tradicionales

      Configuración: En "Configuración" → "Tipo de Dispositivo USB" → selecciona "HID"

      Modo Storage (Almacenamiento)

      ¿Qué hace? El dispositivo aparece como una unidad USB o CD-ROM.

      Casos de uso:

      • Presentar archivos falsos o trampas
      • Distribución de herramientas o malware
      • Simulación de medios de instalación
      • Exfiltración de datos (guardar archivos en la SD)

      Configuración: En "Configuración" → "Clase de Dispositivo USB" → selecciona "Storage"

      Modo NCM (Adaptador de Red)

      ¿Qué hace? El dispositivo se comporta como un adaptador de red USB.

      Casos de uso:

      • Captura de tráfico de red (PCAP)
      • Redireccionamiento de tráfico
      • Ataques man-in-the-middle
      • Monitoreo de comunicaciones

      Configuración: En "Configuración" → "Tipo de Dispositivo USB" → selecciona "NCM"

      🛠️ Resolución de Problemas
      Problemas Comunes

      No puedo conectarme al WiFi:

      • Verifica que el dispositivo haya arrancado completamente (espera 30 segundos)
      • Asegúrate de estar usando la contraseña correcta
      • Reinicia el dispositivo desconectándolo y conectándolo de nuevo

      Los scripts no se ejecutan:

      • Verifica que el archivo tenga extensión .ds
      • Revisa la sintaxis DuckyScript en los logs
      • Asegúrate de que la SD esté correctamente insertada

      La interfaz web no carga:

      • Verifica la dirección IP (debe ser 4.3.2.1:8080)
      • Intenta con otro navegador
      • Limpia la caché del navegador

    • 📋 Guía de Configuración Detallada
      🌐 Configuraciones WiFi
      Modo Punto de Acceso WiFi (wifi-ap-mode)
      Habilita o deshabilita el punto de acceso WiFi del dispositivo. Cuando está activado, el dispositivo crea su propia red WiFi a la que otros dispositivos pueden conectarse. Útil para acceso directo sin necesidad de una red existente.
      Valores: true (habilitado) / false (deshabilitado)
      Nombre del Punto de Acceso WiFi (wifi-ap)
      Define el nombre (SSID) de la red WiFi que creará el dispositivo. Este es el nombre que aparecerá en la lista de redes disponibles en otros dispositivos. Se recomienda usar un nombre que no llame la atención para operaciones sigilosas.
      Valor por defecto: "iPhone14"
      Contraseña WiFi (wifi-pwd)
      Establece la contraseña para el punto de acceso WiFi. Una contraseña fuerte protege el acceso no autorizado al dispositivo. Debe tener al menos 8 caracteres para redes WPA2/WPA3.
      Valor por defecto: "password"
      Estado WiFi al Iniciar (wifi-bootstate)
      Determina si el WiFi se activará automáticamente cuando el dispositivo se encienda. Si está deshabilitado, será necesario activarlo manualmente a través de comandos DuckyScript o la interfaz web.
      Valores: true (activar al inicio) / false (no activar al inicio)
      🔌 Configuraciones USB
      Tipo de Dispositivo USB (usbDeviceType)
      Define qué tipo de dispositivo USB emulará el hardware. Cada tipo ofrece diferentes capacidades y compatibilidad con sistemas operativos.
      • None (0): Sin emulación USB
      • Serial (1): Puerto serie para comunicación y comandos
      • NCM (2): Adaptador de red para acceso a internet
      Clase de Dispositivo USB (usbClassType)
      Especifica la clase USB que determinará cómo el sistema operativo reconoce y maneja el dispositivo. Cada clase tiene diferentes capacidades y drivers asociados.
      • None (0): Sin clase específica
      • HID (1): Dispositivo de interfaz humana (teclado/ratón)
      • Storage (2): Dispositivo de almacenamiento masivo
      USB Vendor ID (VID) (usbDeviceVID)
      Identificador único del fabricante asignado por el USB-IF. El sistema operativo usa este ID para determinar qué driver cargar. Cambiar este valor puede hacer que el dispositivo aparezca como si fuera de otro fabricante.
      Formato: Hexadecimal (ej: 0xCAFE). Valor por defecto: 0xCAFE
      USB Product ID (PID) (usbDevicePID)
      Identificador específico del producto dentro de la línea de productos del fabricante. Junto con el VID, forma la identificación única del dispositivo USB.
      Formato: Hexadecimal (ej: 0x403F). Valor por defecto: 0x403F
      Versión del Protocolo USB (usbVersion)
      Especifica qué versión del estándar USB implementa el dispositivo. Afecta la velocidad de transferencia y las características disponibles.
      Formato: BCD (ej: 0x0101 para USB 1.1, 0x0200 para USB 2.0)
      Versión del Dispositivo USB (usbDeviceVersion)
      Número de versión específico del dispositivo, definido por el fabricante. Se usa para control de versiones del firmware y compatibilidad de drivers.
      Formato: BCD (ej: 0x0101). Valor por defecto: 0x0101
      Fabricante del Dispositivo USB (usbDeviceManufacturer)
      Cadena de texto que identifica al fabricante del dispositivo. Aparece en el administrador de dispositivos del sistema operativo.
      Valor por defecto: "Espressif Systems"
      Descriptor del Producto USB (usbDeviceProductDescriptor)
      Nombre descriptivo del producto que aparecerá en el sistema operativo. Debe ser descriptivo pero no sospechoso para operaciones sigilosas.
      Valor por defecto: "TinyUSB Device"
      Modo Serial USB Sin Procesar (usbSerialRaw)
      Habilita el modo de comunicación serie sin procesamiento adicional. Los datos se transmiten directamente sin formateo ni interpretación de comandos, útil para aplicaciones que requieren control directo.
      Valores: true (modo crudo) / false (modo procesado)
      🤖 Configuraciones del Agente
      Sondeo de Agente Habilitado (agentPollingEnabled)
      Activa el sondeo automático para detectar si hay un agente conectado en la máquina objetivo. El dispositivo enviará periódicamente comandos de verificación para determinar si el agente está activo y respondiendo.
      Valores: true (sondeo activo) / false (sin sondeo automático)
      Tiempo de Sondeo del Agente (agentPollingTime)
      Intervalo en segundos entre intentos de comunicación con el agente. Un valor menor proporciona detección más rápida pero consume más recursos. Un valor mayor es más sigiloso pero puede retrasar la detección.
      Unidad: Segundos. Rango recomendado: 5-60 segundos
      📡 Configuraciones de Red y Captura
      Captura PCAP NCM al Iniciar (pcapNcmOnStart)
      Inicia automáticamente la captura de tráfico de red en formato PCAP cuando se activa el dispositivo NCM (Network Control Model). Útil para análisis de tráfico y monitoreo de red sin intervención manual.
      Valores: true (iniciar captura automática) / false (captura manual)
      🔧 Configuraciones de Sistema
      Retraso de Logger hasta Conexión (loggerdelay)
      Retiene los mensajes de registro en memoria hasta que un debugger o agente se conecte. Esto evita la pérdida de información de depuración importante que ocurre durante el arranque del sistema antes de establecer la comunicación.
      Valores: true (retener logs) / false (mostrar logs inmediatamente)
      💡 Consejos de Configuración:
      • Combina diferentes tipos USB según el ataque: HID para inyección de teclas, Storage para exfiltración de datos
      • Usa nombres WiFi comunes (iPhone, Samsung) para camuflaje en redes públicas
      • Ajusta los VID/PID para emular dispositivos legítimos y evitar detección
      • Habilita el sondeo de agente solo cuando sea necesario para conservar batería
      • El modo serial crudo es útil para comunicación directa con herramientas personalizadas

      📁 Ejemplos Disponibles

      El USB Army Knife incluye una colección completa de scripts de ejemplo que demuestran las diferentes capacidades del dispositivo. Cada ejemplo está diseñado para mostrar técnicas específicas y puede ser personalizado según tus necesidades.

      🗂️ Almacenamiento y Acceso
      covertstorage - Almacenamiento Dual Inteligente
      Usa lógica condicional con FILE_EXISTS() para comportarse diferente según si es la primera conexión. Primera vez: monta SD completa (ATTACKMODE STORAGE). Conexiones posteriores: monta imagen específica (/disk.img) en solo lectura. Deshabilita WiFi/TFT/LED para operaciones sigilosas.
      // Lógica del script:
      IF (FILE_EXISTS() == FALSE) THEN
        CREATE_FILE() + ATTACKMODE STORAGE
      ELSE
        USB_MOUNT_DISK_READ_ONLY /disk.img
      mount_whole_disk_and_run_command - Modo HID + Storage Híbrido
      Configura dispositivo para usar ATTACKMODE HID STORAGE simultáneo. Primero cambia usbDeviceType a NONE persistentemente, luego ejecuta comandos HID (abre PowerShell) mientras monta la SD completa como almacenamiento. Requiere reset inicial para cambiar configuración USB.
      // Flujo real:
      1. SET usbDeviceType = NONE → RESET
      2. ATTACKMODE HID STORAGE
      3. GUI R → "powershell"
      4. SD completa disponible como unidad
      📡 Ataques de Red WiFi
      evilap - Portal Cautivo Apple Falso
      Configura AP "Apple WiFi" con portal cautivo. Usa WIFI_SET_AP con canal 1 y IP 192.168.4.1. Incluye archivo apple.html que simula login de Apple ID para capturar credenciales. Ejecuta commands.txt con lógica de payload personalizable.
      // Configuración real:
      WIFI_SET_AP "Apple WiFi" "" 1 192.168.4.1
      WEB_SET_INDEX_FILE /apple.html
      WIFI_ENABLE_AP
      DUCKY_RUN /commands.txt
      wifi_connect_to_existing_ap - Cliente WiFi + HID
      Se conecta a AP existente usando WIFI_SET_STA con credenciales hardcodeadas "testNetwork"/"testPassword". Después ejecuta HID simulando Alt+Tab y teclas de dirección. Combinación útil para conectividad remota más interacción local.
      // Flujo exacto:
      WIFI_SET_STA "testNetwork" "testPassword"
      WIFI_ENABLE_STA → ALT TAB
      UP UP UP → Interacción automática

      📡 Comandos ESP32 Marauder

      El USB Army Knife integra ESP32 Marauder para ataques WiFi y Bluetooth avanzados. Usa el prefijo ESP32M seguido del comando Marauder:

      🔍 Reconocimiento y Escaneo
      ESP32M scanap - Escanea puntos de acceso WiFi
      ESP32M scansta - Escanea estaciones/clientes conectados
      ESP32M sniffbeacon - Captura tramas beacon
      ESP32M sniffprobe - Captura probe requests
      ESP32M sniffpmkid - Captura PMKIDs para cracking
      ESP32M wardrive - Mapeo geolocalizado de redes
      ⚔️ Ataques WiFi
      ESP32M attack -t deauth - Desautenticación masiva
      ESP32M attack -t beacon -r - Spam de SSIDs aleatorios
      ESP32M attack -t beacon -l - Spam de SSIDs de lista
      ESP32M attack -t probe - Flood de probe requests
      ESP32M attack -t rickroll - Ataque Rickroll WiFi
      ESP32M evilportal - Portal cautivo malicioso
      📱 Ataques Bluetooth
      ESP32M btspamall - Spam BLE combinado
      ESP32M sourapple - Spam emparejamiento Apple
      ESP32M swiftpair - Spam Windows SwiftPair
      ESP32M samsungblespam - Spam Samsung BLE
      ESP32M btwardrive - Wardrive Bluetooth
      🛠️ Control y Configuración
      ESP32M stopscan - Detiene escaneo/ataque activo
      ESP32M channel -s 6 - Establece canal específico
      ESP32M select -a 0,1,2 - Selecciona APs por índice
      ESP32M ssid -a "Red_Test" - Agrega SSID a lista
      ESP32M save -p captura.pcap - Guarda captura
      ESP32M settings -s SavePCAP true - Configura Marauder
      ESP32M clearlist -s - Limpia lista de SSIDs
      ESP32M reboot - Reinicia módulo Marauder
